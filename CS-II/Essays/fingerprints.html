During the early days of the internet, HTTPS was expensive, so it wasn't very common to come across a website with one. However, when the Internet became more prevalent in doing integral tasks, HTTPS was a necesity. After all, it wouldn't be very fun having private info like SSN, addresses, and credit card numbers leaked.

HTTPS makes sure that only the client and the server gets access to the data. One thing that helped in making HTTPS commonplace was Certificate Authorities (CA). Websites can visit CA to create a secure connections after they prove their identity. While going private may seem good for many users, many organizations wanted to access the info.

That is where HTTPS Proxy Appliances come in. The browser would be connected to the HTTPS Proxy Appliance, which would then inspect all the data before establishing a second connection to the remote web server. This practice is known as SSL interception, which can be detected but not prevented.

It can be detected when the crytographic hash (fingerprint) is different from the that of the actual site. When even one bit inside the certificate is changed, the fingerprint would be entirely different. Sometimes, a change in the fingerprint can be the sign of a man-in-the-middle attack.

A man-in-the-middle attack is when someone attempts to steal data by intercepting the connection between the user and the website. There are many ways to prevent a MITM attack, and HTTPS is usually one of the first lines of defense. However, hackers may find a way to bypass it. Sometimes there would also be errors. When it seems like the website is authentic, but the connection is actually being intercepted, it's a false positive. A false positive could happen when the site has multiple certificates.

With the Internet becoming more integral in the last few decades, it begs the question: Is it okay for organizations to sift through people's data? While organizations might have good intentions for doing this, I'm ultimately against this. It is the user that decides what they want to do with their data.
